# Share-safe / PII ban

Цель: исключить попадание персональных данных (PII) и чувствительных данных в репозиторий, PR, Issues, логи CI и артефакты.

## Запрещено (НЕ коммитить / НЕ публиковать / НЕ вставлять в Issues/PR)
- Любые персональные данные (PII):
  - ФИО + связки с контактами/аккаунтами
  - email, телефоны, адреса, паспорта/ID, даты рождения
  - номера карт, банковские реквизиты, ИНН/СНИЛС и т.п.
- Секреты и доступы:
  - токены, API keys, пароли, приватные ключи, cookies, session IDs
  - OAuth refresh tokens, SSH keys, .env с реальными значениями
- Любые данные пользователей/клиентов/третьих лиц, не предназначенные для публикации.
- Скриншоты/логи, где видны реальные контакты, имена, номера, ключи, ссылки с токенами.

## Разрешено (только в обезличенном виде)
- Тестовые/фиктивные данные:
  - `test@example.com`, `+10000000000`, `John Doe`, `REDACTED`
- Конфиги/примеры:
  - `.env.example` без секретов
  - шаблоны ключей/токенов с заглушками
- Логи и отчёты, где чувствительные данные удалены/замаскированы:
  - `***`, `REDACTED`, частичная маска (последние 4 символа)

## Правило редактирования/маскирования
Если есть сомнение — считай это PII/секретом и:
- удали строку полностью, либо
- замени значение на `REDACTED`, либо
- замаскируй так, чтобы восстановление было невозможно.

## Инцидент (если случайно попало в репо)
1) Немедленно удалить из ветки/PR и перепушить фикс (или новый commit, если история защищена).
2) Ротация/отзыв секрета (ключ/токен/пароль) — обязательно.
3) Сообщить в PR кратко: “Removed sensitive data; secret rotated”.
